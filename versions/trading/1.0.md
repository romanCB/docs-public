## 5. Specification

### 5.1 Format


### File Structure

The Cryptogames representation of the trading API is made of a single file. However, parts of the definitions can be split into separate files, at the discretion of the user. This is applicable for `$ref` fields in the specification as follows from the [JSON Schema] definitions.

By convention, the Cryptogames Trading specification file is named `trading.json`.

### Data Types

### Schema

#### Status

	HTTP header: status, description - according to HTTP RFC or provider specific object

##### Fixed fields

Field Name | Type | Size | Description
---|:---|---:|---
code | `string` | 20 | **Required.** error code eg. TRADER_NOT_FOUND, INTERNAL_ERROR
error | `string` | 200 | **Optional.** stack trace
description | `string` | 100 | **Optional.** error description in english language

##### Status object example

HTTP status code = 500

```js
{
	"code": "INTERNAL_ERROR",
	"error": "detailed information about error root cause",
	"description": "Internal server error occured. Please try again in 5 minutes"
}
```

```yaml
code: INTERNAL_ERROR
error: detailed information about error root cause
description: Internal server error occured. Please try again in 5 minutes
```

#### 5.1.1 Get trader account balance

Authentication: Bearer `authToken`

Request object

##### Fixed fields

Field Name | Type | Size | Description
---|:---|---:|---
currency | `string` | 3 | **Required.** according to ISO 4217


Response object

##### Fixed fields

Field Name | Type | Size | Description
---|:---|---:|---
balance | `integer` | int32 | **Required.** total balance for specified currency type
currency | `string` | 3 | **Required.** according to ISO 4217

##### Get account balance object example


```js
{
	"currency": "CHF"
}
```

```yaml
currency: CHF
```

#### 5.1.2 Bet

Authentication: Bearer `authToken`

Request object

##### Fixed fields

Filed Name | Type | Size | Description 
---|:---|---:|---
amount | `integer` | int32 | **Required.** bet amount in pennies, positive integer only
currency | `string` | 3 | **Required.** according to ISO 4217
game | `string` | [1,3] | **Required.** game type symbol eg: WSR, WSB, WF, WB, RFX etc...
roundid | `string` | int32 | **Required.** game round, positive integer only
externalid | `string` | [1,16] | **Required.** unique game identifier for game provider

Response object

<- status - required [200 OK, 5XX errors]
<- error message - optional

##### Bet object example

```js
{
	"amount": 1234,
	"currency": "CHF",
	"game": "WSR",
	"roundid": 1,
	"externalid": "2243"
}
```

```yaml
amount: 1234
currency: CHF
game: WSR
roundid: 1
externalid: 2243
```
